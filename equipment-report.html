1 本資料の目的と概要

資料の目的

本資料は、Genzai（現場資材管理アプリケーション）において新たに設計・実装した備品管理機能および企業管理機能の開発成果を報告するものです。実装した機能の全体像、技術的な設計判断、使用技術を体系的にまとめ、開発の取り組みと成果を共有することを目的としています。

Genzaiとは

Genzaiは、森松工業の現場で使用する資材（工具・消耗品等）を管理するWebアプリケーションです。現場ごとの正確な数量管理、入出庫管理を行い、消耗品以外については備品として台帳管理することで、資材のライフサイクル全体を可視化します。

今回の実装対象

品目管理機能で「台帳管理する」に設定された品目（工具類など現場をまたいで管理する資材）について、実際の備品を1点ずつ登録し、台帳形式でライフサイクル全体を管理する機能を設計・実装しました。具体的には以下の2つの機能群です。

備品管理機能
備品の登録・編集・削除・複写、17項目の検索、5種類の履歴管理（現場使用・移管・メンテナンス・貸出・状態変更）、画像管理（10枚、DnD、カメラ撮影）、添付資料管理（ファイル/URL、各種プレビュー、大容量アップロード）、CSV出力、備品管理番号の自動採番など。
企業管理機能
メーカー・購入先・メンテナンス企業の管理。1企業が複数カテゴリを兼務可能な柔軟な設計。連絡先管理、企業名重複チェック、備品詳細画面からの直接操作、カードグリッドUI。

フロントエンド（Next.js）、バックエンドAPI（Hono）、データベース（PostgreSQL / Prisma）、ストレージ（Supabase Storage）、認証認可（Supabase Auth + IAP）にわたるフルスタック実装です。

新規登録

専用ページ形式（/equipments/new）で登録フォームを提供。登録する情報は以下の通り多岐にわたります。

基本情報（登録項目）

項目 必須 仕様
備品名 必須 最大200文字
品目 必須 品目検索ダイアログで選択（インクリメンタルサーチ+カテゴリフィルタ）
備品管理番号 自動 保管場所選択後に自動採番。プレビュー表示。手動入力も可能
工場管理番号 任意 外部台帳の番号（フリーテキスト）。未入力時は警告色で通知
シリアル番号 任意 製造シリアル番号（最大100文字）
型番 任意 型番・モデル番号（最大100文字）
保管場所 必須 管理工場→管理部署→保管場所の3段カスケード選択
購入区分 必須 購入 / リース / レンタル
重量 任意 数値 + 単位選択（kg/g/t）
寸法 任意 幅・奥行・高さの3値 + 単位選択（cm/m/mm）
メーカー 任意 企業選択ダイアログで選択
購入先 任意 企業選択ダイアログで選択
メンテナンス企業 任意 企業選択ダイアログで選択

品目検索ダイアログの機能

インクリメンタルサーチ
品目名・検索ヒントでデバウンス検索（300ms、2文字以上）。結果を品目カードで表示。
カテゴリフィルタ
トグルバッジで複数カテゴリ同時選択。表示順管理、折りたたみ対応。
台帳管理フィルタ
「台帳管理する品目のみ」「備品区分のみ」のチェックボックス。推奨外品目選択時は確認ダイアログ。
段階読み込み
20件ずつ表示、「もっと見る（残りN件）」で追加ロード。

編集

  • 閲覧モードと編集モードをURLパラメータ（?mode=edit）で切り替え
  • 全ての基本情報・企業情報を編集可能
  • 備品管理番号は編集不可（不変）— 変更不可の旨を画面に明示
  • 品目の変更は可能（管理番号は変わらない旨の警告を表示）

複写機能

複写して編集（単数登録）

  • 既存備品のデータをプリセットした新規登録フォームへ遷移
  • コピー対象: 品目、備品名（「(コピー)」付与）、企業情報、重量、寸法、工場管理番号
  • コピー除外: 管理番号（新規採番）、全履歴、画像、添付資料、シリアル番号

N件複写（一括登録）

  • 1〜10件を指定して一括登録
  • 備品名は「{元の名前}-1」〜「{元の名前}-N」に自動命名
  • 管理番号はそれぞれ自動採番
  • プレビューで作成される備品名と管理番号を事前確認

削除

  • 常に論理削除（deletedAt をセット）。物理削除は不可
  • 削除前に依存チェックを自動実行
  • 削除不可の条件: シートで割り当て中 / 貸出中
  • 依存チェック結果を削除確認ダイアログに表示（削除不可の理由と件数）
  • 「削除済みを表示」切替UIで論理削除済み備品の確認が可能

採番の特性

特性 内容
一意性保証 トランザクション内で排他ロック（FOR UPDATE）を取得して採番
不変 移管で保管場所が変わっても番号は変更しない
欠番 削除された備品の番号は再利用しない

プレビュー用APIと本採番用APIを分離。プレビューはロックなしで軽量に、本採番はトランザクション内で排他ロックを取得して一意性を保証しています。

現在の使用状況（シート連動）

使用状況は現場資材管理シートでの割り当て状態に自動連動して更新されます。台帳からは直接変更できません。

出庫中の場合、使用中の製作番号・現場名・製品名・出庫日・返却予定日をサイドバーに表示します。

10 備品管理 — 画像・写真管理

備品1点につき最大10枚の画像を管理。基本情報の保存とは独立した即時保存方式を採用しています。

アップロード
ドラッグ&ドロップ / ファイル選択 / カメラ撮影の3つの入力方式に対応。JPEG/PNG/WebP、1枚10MBまで。
メイン画像設定
任意の画像をメイン画像として設定可能。一覧テーブルのサムネイルに使用。
DnD並び替え
@dnd-kit による直感的なドラッグ&ドロップ並び替え。表示順を即座に保存。
Swiper表示
Swiper.jsによるスライドショー形式の閲覧。サムネイル行、キーボード操作対応。
カメラ撮影
MediaDevices API（getUserMedia）でPC/iPad/iPhoneすべてでカメラ撮影に対応。
Signed URL
Supabase StorageのSigned URLで画像配信。一覧表示時は一括生成で最適化。

11 備品管理 — 添付資料管理

取扱説明書、保証書、動画マニュアル、メーカーWebサイトなど、備品に関連するあらゆる資料をファイル/URLの両方の形式で管理します。

対応する資料タイプ

種別 タイプ プレビュー/機能
ファイル 画像（JPEG/PNG/WebP/GIF） インラインプレビュー + ダウンロード
動画（MP4/WebM） react-playerによるインライン再生
音声（MP3/WAV/OGG） wavesurfer.jsによるWaveform表示 + 再生
PDF @react-pdf-viewerによるインライン表示
Office系/CSV/テキスト アイコン表示 + ダウンロード
URL Webサイト OGP自動取得（cheerio）でサムネイル・タイトル・説明文を表示
YouTube YouTube Data API v3でサムネイル取得 + react-playerで再生

アップロード機能

通常アップロード
20MB以下のファイルは通常のHTTPアップロード。react-dropzoneでD&D対応。5秒以内の完了を目標。
TUS Resumable Upload
20MB超の大容量ファイルはtus-js-clientによるリジューマブルアップロード。中断時に再開可能。プログレスバーで進捗表示。
サムネイル自動生成
バックエンドでsharpを使用して画像サムネイルを自動生成。一覧表示の軽量化。
OGP取得
URL登録時にcheerioでOGP情報（タイトル、説明、画像）を自動取得。

資料管理UI

  • 2カラム構成: 左に一覧パネル、右に詳細パネル（プレビュー表示）
  • DnD並び替え対応（@dnd-kit）。並び順は即時保存
  • 閲覧モードの一覧行ホバー時に説明をTooltipで表示
  • 操作ボタン: ダウンロード、上移動、下移動、削除、外部リンク
  • エラーメッセージは日本語で統一（400: ファイル不正、409: 上限到達のステータス別分岐）
  • 品目管理と備品管理で共通コンポーネントを再利用（API層のみ差替え）
仕様 内容
最大件数 20件
保存先 Supabase Storage（equipments/{equipmentId}/attachments/{attachmentId}.{ext}）
保存タイミング 即時保存（基本情報と独立）

12 企業管理

備品に関連するメーカー、購入先、メンテナンス企業を管理する独立した機能です。1企業が複数のカテゴリを兼ねることが可能な柔軟な設計。

企業登録

項目 必須 仕様
企業名 必須 最大200文字。重複チェック付き（完全一致で警告、登録はブロックしない）
カテゴリ 必須 メーカー/購入先/メンテナンス企業から複数選択可能。1つ以上必須
備考 任意 最大500文字
連絡先 任意 複数登録可能。代表連絡先を1つ設定

連絡先管理

項目 仕様
担当者名 最大100文字
メールアドレス メールアドレス形式バリデーション
電話番号 最大20文字
代表フラグ 代表連絡先を1つ設定。一覧カードや備品詳細で代表者情報を表示

企業ページ（一覧・検索）

カードグリッド表示
PC 3列、タブレット 2列、モバイル 1列のレスポンシブグリッド。企業情報をカード形式で一覧。
インライン検索
企業名（部分一致）、カテゴリ（トグルボタン複数選択）、担当者名（部分一致）。
URL連動ダイアログ
新規登録・編集・複写のダイアログ表示状態をnuqsでURL管理。直接URLアクセスで該当ダイアログを表示。
企業名重複チェック
フォーカスアウト時にAPI呼び出しで完全一致チェック。同名企業の散乱を防止。

備品詳細画面との連携

  • 備品詳細画面で企業情報（メーカー・購入先・メンテ企業）をインライン表示
  • 代表連絡先の担当者名・電話番号・メールアドレスを表示
  • 備品詳細画面から直接、企業の選択・変更・解除が可能
  • 企業が未登録の場合、「企業を選択して登録」ダイアログと「企業ページで新規登録」への導線を提供

複写・削除

  • 複写して作成: 企業名（「(コピー)」付与）、カテゴリ、備考、連絡先をコピー
  • N件複写: 1〜10件指定、企業名は「{名前}-1」〜「{名前}-N」に自動命名
  • 論理削除: 備品から参照されている場合は削除不可（依存チェック結果を表示）
  • 「削除済みを表示」切替UIを提供

14 非機能要件への対応

パフォーマンス最適化

項目 対応
初回レンダリング Server Componentでデータ取得 → TanStack Query initialDataに設定（SSR最適化）
備品一覧クエリ 必要な列のみSELECT、JOINの最適化、カウントクエリ分離。N+1問題を回避
画像配信 Signed URLの一括生成（一覧表示時）。1時間有効
検索レスポンス 1秒以内（通常データ量）。適切なインデックス設計
アップロード 10MB以下: 5秒以内 / 20MB超: TUS resumable + プログレスバー
管理番号採番 トランザクション内で排他ロック（FOR UPDATE）による一意性保証
